Dify 是一个用于开发大语言模型（LLM）应用的开源平台。其直观的界面结合了智能体 AI 工作流、RAG（检索增强生成）管道、智能体能力、模型管理、可观测性功能等，让您能够快速从原型转向生产环境。

## 快速开始

> 在安装 Dify 之前，请确保您的机器满足以下最低系统要求：
>
> - CPU >= 2 核
> - RAM >= 4 GiB

<br/>

启动 Dify 服务器最简单的方法是通过 [Docker Compose](docker/docker-compose.yaml)。在使用以下命令运行 Dify 之前，请确保您的机器上已安装 [Docker](https://docs.docker.com/get-docker/) 和 [Docker Compose](https://docs.docker.com/compose/install/)：

```bash
cd dify           # 进入 dify 目录
cd docker         # 进入 docker 目录
cp .env.example .env  # 复制环境变量示例文件
docker compose up -d  # 使用 Docker Compose 启动服务（后台运行）
```

运行后，您可以在浏览器中访问 [http://localhost/install](http://localhost/install) 来访问 Dify 仪表板并开始初始化过程。

#### 寻求帮助

如果您在设置 Dify 时遇到问题，请参考我们的 [常见问题解答（FAQ）](https://docs.dify.ai/getting-started/install-self-hosted/faqs)。如果您仍有问题，请联系[社区和我们](#社区--联系方式)。

> 如果您想为 Dify 做贡献或进行额外开发，请参考我们的[从源代码部署指南](https://docs.dify.ai/getting-started/install-self-hosted/local-source-code)

## 核心功能

**1. 工作流（Workflow）**：
在可视化画布上构建和测试强大的 AI 工作流，利用以下所有功能以及更多功能。

**2. 全面的模型支持**：
与来自数十个推理提供商和自托管解决方案的数百个专有/开源 LLM 无缝集成，涵盖 GPT、Mistral、Llama3 以及任何兼容 OpenAI API 的模型。支持的模型提供商完整列表可以在[这里](https://docs.dify.ai/getting-started/readme/model-providers)找到。

![providers-v5](https://github.com/langgenius/dify/assets/13230914/5a17bdbe-097a-4100-8363-40255b70f6e3)

**3. Prompt IDE（提示词集成开发环境）**：
直观的界面，用于精心制作提示词、比较模型性能，以及为基于聊天的应用程序添加文本转语音等附加功能。

**4. RAG 管道（检索增强生成管道）**：
广泛的 RAG 能力，涵盖从文档摄取到检索的所有内容，开箱即用地支持从 PDF、PPT 和其他常见文档格式中提取文本。

**5. 智能体能力**：
您可以基于 LLM 函数调用或 ReAct 定义智能体，并为智能体添加预构建或自定义工具。Dify 为 AI 智能体提供了 50 多个内置工具，例如 Google 搜索、DALL·E、Stable Diffusion 和 WolframAlpha。

**6. LLMOps（大语言模型运维）**：
监控和分析应用程序日志和性能随时间的变化。您可以根据生产数据和注释持续改进提示词、数据集和模型。

**7. 后端即服务（Backend-as-a-Service）**：
Dify 的所有产品都提供相应的 API，因此您可以轻松地将 Dify 集成到您自己的业务逻辑中。

## 高级设置

### 自定义配置

如果您需要自定义配置，请参考我们的 [.env.example](docker/.env.example) 文件中的注释，并更新 `.env` 文件中的相应值。此外，根据您的具体部署环境和需求，您可能需要调整 `docker-compose.yaml` 文件本身，例如更改镜像版本、端口映射或卷挂载。进行任何更改后，请重新运行 `docker-compose up -d`。您可以在[这里](https://docs.dify.ai/getting-started/install-self-hosted/environments)找到可用环境变量的完整列表。

#### 自定义建议问题

您现在可以自定义"回答后建议问题"功能，以更好地适应您的使用场景。例如，要生成更长、更具技术性的问题：

```bash
# 在您的 .env 文件中
SUGGESTED_QUESTIONS_PROMPT='请帮助我预测开发者最可能问的五个技术后续问题。重点关注实现细节、最佳实践和架构考虑。保持每个问题在 40-60 个字符之间。输出必须是 JSON 数组：["question1","question2","question3","question4","question5"]'
SUGGESTED_QUESTIONS_MAX_TOKENS=512  # 设置最大 token 数为 512
SUGGESTED_QUESTIONS_TEMPERATURE=0.3  # 设置温度参数为 0.3（控制随机性）
```

有关详细示例和使用说明，请参阅[建议问题配置指南](docs/suggested-questions-configuration.md)。
